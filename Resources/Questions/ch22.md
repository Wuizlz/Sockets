# Networking Questions and Answers

This document collects the self-check questions and review exercise answers you provided into a single study reference.

## Self-Check Questions

### 1. What is the difference between an IP address and a domain name?
An IP address is a numerical address consisting of four or sixteen bytes. A domain name is an alphanumeric name associated with an IP address.

### 2. Why do some streaming media services not use TCP?
TCP is reliable but somewhat slow. For real-time sound or image streaming, a little data loss is acceptable, but late data is not useful.

### 3. Why don't you need to know about HTTP when you use a web browser?
The browser translates typed URLs and clicks into HTTP commands and sends them to the proper web servers.

### 4. Why is it important that you don't make typing errors when you type HTTP commands in Telnet?
Some Telnet implementations send every keystroke, including backspace, so a mistyped sequence may not be recognized as a valid command by the server.

### 5. What happens if you call WebGet with a nonexistent resource, such as `wombat.html` at `horstmann.com`?
The program connects to the server, sends the GET request, and prints the error message returned by the server.

### 6. How do you open a socket to read e-mail from the POP server at `e-mail.sjsu.edu`?

```java
Socket s = new Socket("e-mail.sjsu.edu", 110);
```

### 7. Why didn't we choose port 80 for the bank server?
Port 80 is the standard HTTP port. If a web server is already using it, you cannot open the bank server on that same port.

### 8. Can you read data from a server socket?
No. A `ServerSocket` only waits for a connection and returns a regular `Socket` when a client connects. You read the client's data from that returned `Socket`.

### 9. Why is it better to use a `URLConnection` instead of a socket when reading data from a web server?
`URLConnection` understands HTTP, so you do not have to manually assemble requests or analyze response headers.

### 10. What happens if you use the `URLGet` program to request an image?
The image bytes are printed to the console, but they appear as random gibberish.

## Review Exercises

### R22.1 What is the IP address of the computer that you are using at home? Does it have a domain name?
Your home computer has whatever IP address your ISP or router assigns it. It may or may not have a domain name; domain names are easier-to-remember names that DNS translates into IP addresses.

### R22.2 Can a computer somewhere on the Internet establish a network connection with the computer at your home? If so, what information does the other computer need to establish the connection?
Yes, if your home computer is reachable. The other computer needs the destination IP address and the destination port number. A TCP connection also involves the sender's IP address and sender's port number.

### R22.3 What is a port number? Can the same computer receive data on two different ports?
A port number identifies the specific program or service on a computer. Yes, the same computer can receive data on different ports at the same time.

### R22.4 What is a server? What is a client? How many clients can connect to a server at one time?
A server waits for requests and provides a service. A client initiates the request. A server can handle multiple clients, often by creating a separate thread per connection.

### R22.5 What is a socket? What is the difference between a `Socket` object and a `ServerSocket` object?
A socket encapsulates a TCP connection. A `Socket` is used for the actual communication. A `ServerSocket` listens for incoming connections and yields a regular `Socket` when a client connects.

### R22.6 Under what circumstances would an `UnknownHostException` be thrown?
When the socket constructor cannot find or resolve the host name.

### R22.7 What happens if the `Socket` constructor's second argument is not the same as the port number at which the server waits for connections?
The client will not connect to the intended service. The connection may fail, or it may connect to some other service if something else is listening on that port.

### R22.8 When a socket is created, which Internet address is used?
The address of the computer to which you want to connect.

### R22.9 What is the purpose of the `accept` method of the `ServerSocket` class?
It waits for an incoming client connection and returns a `Socket` for communicating with that client.

### R22.10 After a socket establishes a connection, which mechanism will your client program use to read data from the server computer?
You will use an `InputStream` obtained from the `Socket`.

### R22.11 Why is it not common to work directly with the `InputStream` and `OutputStream` objects obtained from a `Socket` object?
Because they work with raw bytes. For text protocols, it is more convenient to wrap them in higher-level tools such as `Scanner` and `PrintWriter`.

### R22.12 When a client program communicates with a server, it sometimes needs to flush the output stream. Explain why.
Because a `PrintWriter` buffers output. Flushing forces the complete command to be sent immediately instead of waiting in the buffer.

### R22.13 What is the difference between HTTP and HTML?
HTTP is the communication protocol between web browsers and web servers. HTML is the document format that describes the structure and content of a web page.

### R22.14 Try out the `HEAD` command of the HTTP protocol. What command did you use? What response did you get?
Example command:

```text
HEAD / HTTP/1.1
Host: horstmann.com
```

Expected response: the server returns the HTTP status line and headers only, without the page body. `HEAD` requests only the header information of an item.

### R22.15 Connect to a POP server that hosts your e-mail and retrieve a message. Provide a record of your session.
A typical POP session looks like:

```text
USER yourUsername
PASS yourPassword
STAT
RETR 1
QUIT
```

The sample POP exchange also includes replies such as `+OK` and may include commands like `DELE 1`.

### R22.16 How can you communicate with a web server without using sockets?
By using `URLConnection` or `HttpURLConnection`.

### R22.17 What is the difference between a `URL` instance and a `URLConnection` instance?
A `URL` represents the address of the resource. A `URLConnection` represents the opened connection to that resource and lets you retrieve data and metadata.

### R22.18 What is a URL? How do you create an object of class `URL`? How do you connect to a URL?
A URL is a Uniform Resource Locator, a pointer to an information resource on the web.

```java
URL u = new URL("http://horstmann.com/index.html");
URLConnection connection = u.openConnection();
InputStream in = connection.getInputStream();
```
